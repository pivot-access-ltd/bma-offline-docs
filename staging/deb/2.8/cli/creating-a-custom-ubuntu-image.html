Creating a custom ubuntu image

Introduction

MAAS supports deploying custom DD or TGZ images. Canonical provides both lp:maas-deb-2-8-cliimage-deb-2-8-clibuilder and gh:canonical/packer-deb-2-8-climaas to support creating custom images however these tools do not currently support Ubuntu. Instead Canonical suggests customising Ubuntu using cloud-deb-2-8-cliinit user_data or Curtin preseed data.

Why customised Ubuntu deployments aren’t supported

When the MAAS stream is generated by lp:maas-deb-2-8-cliimages it starts by downloading the base SquashFS rootfs from cloud-deb-2-8-cliimages.ubuntu.com that is used for all clouds. The SquashFS does not contain a kernel so lp:maas-deb-2-8-cliimages mounts the SquashFS with an overlay and chroots in. It then installs a kernel and extra initramfs scripts from the cloud-deb-2-8-cliinitramfs-deb-2-8-clirooturl and cloud-deb-2-8-cliinitramfs-deb-2-8-clicopymods packages to allow network booting. Once everything is installed the kernel and newly generated initramfs are pulled out of the overlay and everything is unmounted. lp:maas-deb-2-8-cliimages provides the unmodified SquashFS, installed kernel, and generated initramfs as separate files on images.maas.io.

MAAS uses the kernel, initramfs, and SquashFS to perform network booting which allows commissioning, testing, and deployments. When deploying Ubuntu MAAS uses the same version of Ubuntu to network boot into and perform the deployment. This ensures there are no compatibility issues. Other operating systems use the Ubuntu version selected for the ephemeral environment for deployment.

Currently MAAS only allows custom images to be loaded as a single TGZ or DD.GZ, there is no way to upload a kernel, initrd, and rootfs as separate files. Even if MAAS was modified to allow the kernel, initrd, and rootfs as separate files MAAS requires cloud-deb-2-8-cliinitramfs-deb-2-8-clicopymods and cloud-deb-2-8-cliinitramfs-deb-2-8-clirooturl to be included in the initrd. It is difficult for MAAS to detect if these scripts are missing and even harder for users to debug if they are missing.

Warnings on creating a custom Ubuntu image

  1. Custom images are always deployed with the ephemeral operating system. This can cause hard to debug errors. For example CentOS 6 can only be deployed by Ubuntu Xenial due to advances in the ext4 filesystem.
  2. MAAS will still install and configure the GA kernel. If your custom image contains a kernel it most likely won’t be used. MAAS will not allow you to select which kernel(GA, HWE, lowlatency, etc) when deploying a custom Ubuntu image.
  3. All GNU/Linux custom images should be created as a TGZ to enable storage customisation. When a DD.GZ is used MAAS/Curtin simply writes the file to the filesystem.

How to create a custom Ubuntu image for MAAS

Note: LXD may prevent device files from being created when extracting the rootfs, it is suggested to do this on metal or on a VM:

  1. Download the rootfs from images.maas.io wget http://cloud-deb-2-8-cliimages.ubuntu.com/focal/current/focal-deb-2-8-cliserver-deb-2-8-clicloudimg-deb-2-8-cliamd64-deb-2-8-cliroot.tar.xz
  2. Create a work directory mkdir /tmp/work

  3. Extract the rootfs cd /tmp/work sudo tar xf ../focal-deb-2-8-cliserver-deb-2-8-clicloudimg-deb-2-8-cliamd64-deb-2-8-cliroot.tar.xz

    or

unsquashfs ../focal-deb-2-8-cliserver-deb-2-8-clicloudimg-deb-2-8-cliamd64-deb-2-8-cliroot.squash 4. Setup chroot sudo mount -deb-2-8-clio bind /proc /tmp/work/proc sudo mount -deb-2-8-clio bind /dev /tmp/work/dev sudo mount -deb-2-8-clio bind /sys /tmp/work/sys sudo mv /tmp/work/etc/resolv.conf /tmp/work/etc/resolv.conf.bak sudo cp /etc/resolv.conf /tmp/work/etc/ 5. Chroot in sudo chroot /tmp/work /bin/bash 6. Customise image apt update apt dist-deb-2-8-cliupgrade apt install emacs-deb-2-8-clinox jq tree apt autoremove 7. Exit chroot and unmount binds exit sudo umount /tmp/work/proc sudo umount /tmp/work/dev sudo umount /tmp/work/sys sudo mv /tmp/work/etc/resolv.conf.bak /tmp/work/etc/resolv.conf 8. Create TGZ sudo tar -deb-2-8-cliczf /tmp/focal-deb-2-8-clicustom.tgz -deb-2-8-cliC /tmp/work . 9. Upload it to MAAS Note: Ubuntu release names and versions are reserved maas $PROFILE boot-deb-2-8-cliresources create name='custom/focal-deb-2-8-clicustom' title='Ubuntu 20.04 Custom Image' architecture='amd64/generic' filetype='tgz' content@=focal-deb-2-8-clicustom.tgz 10. Configure and deploy as normal
