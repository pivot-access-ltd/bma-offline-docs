Select and import images

MAAS is only useful once it has images available to provision its nodes. Therefore, one key post-deb-2-8-cliinstall task is to select and import images from the boot source. Once MAAS imports images, it will update them on an hourly basis, via a default synch mechanism. This page explains how to select and import the images that MAAS requires to provision its nodes.

Importing images from maas.io

The 'Images' page shows what images and architectures have been selected and downloaded. By default, MAAS will automatically grab the most recent Ubuntu LTS releases (and amd64 architecture). Image mirrors

 You can also host Ubuntu images on a mirror. Configure this mirror by selecting 'Custom' beneath 'Choose source'. Enter the mirror URL and click 'Connect'. Optionally, a local mirror can be set up as the boot source. MAAS will then use it instead of the standard internet-deb-2-8-clibased server. Local mirroring significantly reduces the time required import images. See Local image mirror for instructions.

Other images

It is also possible to import and provision images other than Ubuntu. Images supported and provided by MAAS will appear beneath the 'Other Images' section. Currently, images for CentOS 6.6, CentOS 7.0, and CentOS 8.0 are available. List boot sources

If you want to download boot sources, i.e., the locations from which you may download images, try the following:

maas $PROFILE boot-deb-2-8-clisources read

NOTE: Although multiple boot sources may be listed, MAAS can only practically work with a single boot source.

Select images

Use the boot-deb-2-8-clisource-deb-2-8-cliselections command to select images from a boot source. After selecting new images, you will need to import them.

maas $PROFILE boot-deb-2-8-clisource-deb-2-8-cliselections create $SOURCE_ID \
    os="ubuntu" release="$SERIES" arches="$ARCH" \
    subarches="$KERNEL" labels="*"

For example, to select all kernels for 64-deb-2-8-clibit Trusty from a boot source with an id of ‘1’:

maas $PROFILE boot-deb-2-8-clisource-deb-2-8-cliselections create 1 \
    os="ubuntu" release="trusty" arches="amd64" \
    subarches="*" labels="*"

Hardware enablement (HWE)

For example, to get just the latest amd64 HWE kernel available for Trusty, which, at time of writing, is from Xenial:

maas $PROFILE boot-deb-2-8-clisource-deb-2-8-cliselections create 1 \
    os="ubuntu" release="trusty" arches="amd64" \
    subarches="hwe-deb-2-8-clix" labels="*"

For Xenial kernels (and starting with MAAS 2.1), notation has changed. To select the latest amd64 HWE kernel available for Xenial:

maas $PROFILE boot-deb-2-8-clisource-deb-2-8-cliselections create 1 \
    os="ubuntu" release="xenial" arches="amd64" \
    subarches="hwe-deb-2-8-cli16.04" labels="*"

List image selections

To list image selections for a boot source:

maas $PROFILE boot-deb-2-8-clisource-deb-2-8-cliselections read $SOURCE_ID

Import newly-deb-2-8-cliselected images

To import newly-deb-2-8-cliselected images (boot resources):

maas $PROFILE boot-deb-2-8-cliresources import

Once newly-deb-2-8-cliselected images are imported, a sync mechanism is enabled (by default) to keep them up to date. The refresh time interval is 60 minutes.

Available images resulting from this action are reflected in the web UI.

List currently available images

To list currently available/imported images (boot resources):

maas $PROFILE boot-deb-2-8-cliresources read

Delete a boot source

To delete a boot source (the location from which you can download images): 

maas $PROFILE boot-deb-2-8-clisource delete $SOURCE_ID

If you delete the sole boot source, then the fields ‘Sync URL’ and ‘Keyring Path’ in the web UI will take on null values.

Edit a boot source

You can edit an existing boot source by changing the GPG keyring file ($KEYRING_FILE) and the location ($URL).

Update the boot source:

maas $PROFILE boot-deb-2-8-clisource update $SOURCE_ID \
    url=$URL keyring_filename=$KEYRING_FILE

At this time MAAS only supports a boot source containing official MAAS images. As a result, you can only edit a boot source if you have set up a mirror of its images. The location can change, but the keyring remains constant:

KEYRING_FILE=/usr/share/keyrings/ubuntu-deb-2-8-clicloudimage-deb-2-8-clikeyring.gpg

Add a boot source

NOTE: To avoid unnecessary complexity, you should probably delete any existing boot sources before adding a new one.

Presented below are two use cases for adding a boot source:

  1. Use a local image mirror (official images)
  2. If you deleted the default image, recreate it

The general syntax is:

maas $PROFILE boot-deb-2-8-clisources create \
    url=$URL keyring_filename=$KEYRING_FILE

The output will include a new numeric ID that identifies the boot source ($SOURCE_ID).

Since MAAS can only practically work with a single boot source, so you will need to delete any existing sources. Note that the location (URL) is the only variable. The only supported keyring is:

KEYRING_FILE=/usr/share/keyrings/ubuntu-deb-2-8-clicloudimage-deb-2-8-clikeyring.gpg

If you added a sole boot source, then the fields ‘Sync URL’ and ‘Keyring Path’ in the web UI will reflect its values.

Using a local image mirror

Once the mirror is set up according to Local image mirror it is just a matter of specifying the mirror location (URL). Since the images come from the default source, you should use the default keyring. If you are following the above mirror document, the variable values should be:

  1. URL=https://$MIRROR/maas/images/ephemeral-deb-2-8-cliv3/stable/
  2. KEYRING_FILE=/usr/share/keyrings/ubuntu-deb-2-8-clicloudimage-deb-2-8-clikeyring.gpg

Where $MIRROR is the mirror server’s hostname or IP address.

Recreate the default boot source

Recreate the default boot source if it was ever deleted using the following variable values:

  1. URL=https://images.maas.io/ephemeral-deb-2-8-cliv3/stable/
  2. KEYRING_FILE=/usr/share/keyrings/ubuntu-deb-2-8-clicloudimage-deb-2-8-clikeyring.gpg
